# Project Standard

本规范适用于项目中的 JavaScript、TypeScript、React 组件和样式文件。开发时应优先保持实现简洁、语义清晰，并与项目既有风格一致。

## 通用约束

1. 仅在代码意图、边界条件或复杂逻辑不易从实现中直接看出时添加注释；避免用注释重复描述代码本身。
2. 样式优先使用 UnoCSS 原子类实现。只有在需要复用复杂样式、处理全局样式，或原子类无法清晰表达时，才补充 CSS 文件。
3. 组件开发应优先复用项目已有组件与设计规范。确需新增通用 UI 组件时，优先使用 `shadcn` skill 创建，并保持命名、结构和交互风格一致。
4. 不能引入不合理的设计，代码要尽量简洁清晰，行为要可预测，路径要干净
5. 工作目录在：`web`

## 样式约束

1. 页面视觉应保持简约、现代和克制，参考 Apple 官网在留白、层级、动效和质感上的处理方式。
2. 优先保证内容可读性和交互清晰度，避免过度装饰、复杂动效和高噪声视觉元素。
